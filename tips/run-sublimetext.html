Sublime Text 3をコマンドラインから起動する

コーディングを行う際にSublime Text 3を使うが、これを起動するのが面倒くさい。
terminal上のコマンドラインでSublime Text 3を起動できるようにする。

コマンドラインからの起動

ターミナルなどのアプリケーションはコマンドラインからファイルを実行することでも起動できる。
Sublime Text 3の実行ファイルは「/Applications/Sublime Text.app/Contents/SharedSupport/bin/」ディレクトリにあるsublというファイルなので、これを実行するとSublime Text 3を起動できる。

$ /Applications/Sublime\ Text.app/Contents/SharedSupport/bin/subl

※バックスラッシュ(\)は空白文字を識別するために挿入している。

しかし、毎回こんな長いコマンドを打つのは面倒だし、現実的ではない。

sublだけで起動できるようにする

sublを実行するためにフルパスを指定する必要があるのはシステムのパスが通っていないため。
そのため、環境変数のPATHにsublが存在するディレクトリを追加すればsublと打つだけで起動できる。
しかし、個々のアプリケーションのためにシステムの環境変数を変更することは望ましくない。

/usr/local/binにシンボリックリンクを作成する

そこで/usr/local/binディレクトリにsublファイルを指定したシンボリックリンクを作る。
/usr/local/binディレクトリはユーザがインストールしたソフトウェアのプログラムを配置するためのもので、
デフォルトでPATHが通っている。

$ echo $PATH
/Users/tech-camp/.rbenv/shims:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin

以下のコマンドで/usr/local/binディレクトリにSublime Text 3を起動するためのシンボリックリンクを作成する。

$ ln -s /Applications/Sublime\ Text.app/Contents/SharedSupport/bin/subl /usr/local/bin/

作成したシンボリックリンクを確認。

$ ls -l /usr/local/bin | grep subl
lrwxr-xr-x 1 tech-camp admin 62 12 3 11:54 subl -> /Applications/Sublime Text.app/Contents/SharedSupport/bin/subl

これは/usr/local/binディレクトリにあるsublを実行すると、
アプリケーションフォルダ配下のsublが実行されるという意味。
そして、/usr/local/binディレクトリにはパスが通っているのでsublと打つだけでSublime Text 3を起動できる。

$ subl ( = Sublime Text 3が起動する)

ディレクトリを開いた状態で起動する

sublの引数にファイルを指定すると、そのファイルを開いた状態でSublime Text 3を起動できる。

$ subl <開きたいファイル名>

また、sublの引数にディレクトリを指定するとサイドバーにそのディレクトリが表示された状態でSublime Text 3が起動する。
例えば、Railsアプリケーションのトップディレクトリを開きたい時は、ターミナル上でそのディレクトリに移動し、カレントディレクトリを引数に与えてsublを実行すれば良い。

$ cd ~/projects/pictweet
$ subl .

これでcommand + Oでディレクトリを開く必要がなくなる。とても便利。
